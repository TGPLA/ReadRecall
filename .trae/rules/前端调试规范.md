# 前端调试规范

## 核心原则

- **TypeScript 编译通过 ≠ 运行时没问题**
- TypeScript 只做静态类型检查，不模拟代码执行顺序
- `useCallback` 依赖数组在运行时求值，定义顺序很重要

## 调试流程（严格执行）

```
1. 构建成功 →
2. 启动开发服务器 →
3. 打开浏览器 →
4. 检查控制台是否有错误 →
5. 定位并修复
```

## 禁止行为

- 禁止仅凭构建成功就认为问题已解决
- 禁止忽略白屏问题而不检查控制台
- 禁止在没有看到实际运行效果前结束调试

## 常见运行时错误

- `Cannot access 'xxx' before initialization`：变量定义顺序问题
- 白屏但无报错：检查组件渲染逻辑、异步加载状态
- 网络请求失败：检查 API 代理配置、后端服务状态
