# 全局语言规范

## 触发条件

任何代码编写、注释、文档创建、对话交流场景

## 核心原则

**全局语言统一使用中文**，包括但不限于：

| 场景 | 要求 |
|------|------|
| 代码注释 | 使用中文 |
| 变量命名 | 使用中文（如 `shuMing`） |
| 函数命名 | 使用中文（如 `huoQuPeiZhi`） |
| 文件命名 | 使用中文拼音（如 `ShuJiTiShiCiPeiZhi.ts`） |
| 对话交流 | 使用中文 |
| 思考过程 | **必须使用中文** |
| 文档编写 | 使用中文 |

## 允许例外的情况

1. **第三方库 API**：保留英文原名（如 `fetch`, `axios`）
2. **技术术语**：保留英文技术术语（如 `API`, `JSON`）
3. **国际标准**：保留国际标准缩写（如 `URL`, `HTTP`）
4. **测试代码**：测试场景可使用英文描述

## 命名优先级

1. **首选**：中文拼音全拼（如 `shuMing`）
2. **次选**：中文缩写（如 `sm` 不推荐）
3. **禁止**：纯英文变量名（如 `name`）

## 关于思考过程的特殊强调

**思考过程（reasoning/thought）必须使用中文输出**，这是最容易被忽视但同样重要的场景：

| 禁止行为 | 正确做法 |
|----------|----------|
| ❌ 思考时用英文，只把最终回复翻译成中文 | ✅ 从思考到回复全程使用中文 |
| ❌ 调用英文技能后被影响使用英文思考 | ✅ 无论调用什么技能，思考保持中文 |
| ❌ 在 `<thinking>` 或 `<reasoning>` 标签内使用英文 | ✅ 所有内部思考文字都用中文 |

## 检查方式

- 代码审查时检查注释和命名语言
- 新文件创建时验证头部注释语言
- **自我检查时确认思考过程使用中文**
