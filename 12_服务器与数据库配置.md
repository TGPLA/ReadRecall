## 数据库连接方式

### 本地开发（SSH 隧道）

#### 第一步：建立 SSH 隧道
```bash
ssh -i "$env:USERPROFILE\.ssh\id_rsa" -f -N -L 3307:127.0.0.1:3306 root@linyubo.top
```

#### 第二步：配置后端 .env
```bash
DB_HOST=127.0.0.1
DB_PORT=3307
DB_USER=root
DB_PASSWORD=mysql_h8TCKC
DB_NAME=reading_reflection
```

**注意**：当前配置使用 `root` 用户连接远程 MySQL，通过 SSH 密钥认证。

### 服务器生产环境

#### Docker 容器网络
```bash
DB_HOST=1Panel-mysql-OHb5  # MySQL 容器名称
DB_PORT=3306
DB_USER=readrecall
DB_PASSWORD=&lt;数据库密码&gt;
DB_NAME=reading_reflection
```

## SSH 隧道配置

### Windows PowerShell
```powershell
# 建立隧道
ssh -f -N -L 3307:127.0.0.1:3306 root@linyubo.top

# 查看隧道进程
Get-Process | Where-Object {$_.ProcessName -eq "ssh"}

# 关闭隧道
Stop-Process -Name "ssh"
```

### macOS / Linux
```bash
# 建立隧道
ssh -f -N -L 3307:127.0.0.1:3306 root@linyubo.top

# 查看隧道进程
ps aux | grep ssh

# 关闭隧道
pkill -f "ssh -f -N -L 3307"
```


#### 🤖 自动重连方案（推荐）

项目已提供自动重连脚本，可避免手动维护 SSH 隧道：

**文件位置：**
- `SSH-Tunnel.ps1` - 自动重连脚本
- `ssh_config` - SSH 配置（含心跳保活）

**使用方法：**

1. 复制 SSH 配置（首次需要）：
   ```powershell
   copy e:\阅读回响\ssh_config C:\Users\Administrator\.ssh\config
   ```

2. 启动自动隧道管理器：
   ```powershell
   powershell -ExecutionPolicy Bypass -File "e:\阅读回响\SSH-Tunnel.ps1"
   ```

**功能说明：**
- 每 30 秒检测端口 3307 状态
- 端口断开时自动重新建立 SSH 隧道
- 配合 ssh_config 中的 `ServerAliveInterval 60` 防止 idle 断连

## 安全建议

1. **使用强密码**：数据库密码和 JWT 密钥应使用随机生成的强密码
2. **定期轮换**：定期更换数据库密码和 JWT 密钥
3. **最小权限**：数据库用户仅授予必要的权限
4. **不要提交**：永远不要将 `.env` 文件提交到 Git 仓库
5. **备份配置**：安全地备份生产环境的配置文件
