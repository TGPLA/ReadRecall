### 选词弹窗交互优化（2026-04-04）

**完成内容**：
修复 EPUB 阅读器选词弹窗"点击一次选中效果消失，再点一次弹窗才消失"的问题。参考 GitHub 项目 Alexandria（Tauri+epub.js 阅读器）的 Annotation 持久化方案。

**根本原因分析**：

1. **浏览器原生行为**：点击空白处时，浏览器立即清除文本选区（选中效果消失）
2. **事件时序冲突**：iframe 内 `mousedown` 监听器在 10ms 后就清除临时标注和弹窗

**修复方案**：

| 改动 | 之前 | 之后 |
|------|------|------|
| 选中效果 | 浏览器原生选区 | epub.js 临时 annotation（蓝色半透明） |
| iframe 监听 | mousedown + 10ms 延迟 | click + 无延迟 |
| 状态读取 | 闭包中的 showMenu | showMenuRef.current（useEffect 同步） |
| 弹窗关闭 | mousedown 外部检测 | click 外部检测 |

**修改文件**：
- [useEPUBReaderShiJian.ts](src/features/books/hooks/useEPUBReaderShiJian.ts) — 新增临时标注系统、showMenuRef、click 事件处理

- [useEPUBReaderHuoChuLi.ts](src/features/books/hooks/useEPUBReaderHuoChuLi.ts) — handleShowMenu 中添加临时标注清理

### 高亮颜色多色支持修复（2026-04-04）

**完成内容**：
修复 EPUB 阅读器高亮标记所有颜色都显示为黄色的问题。

**根本原因**：

epub.js 的 `iframe.js` 中 `highlight()` 方法使用 `Object.assign({fill: "yellow", ...}, styles)` 合并样式。之前代码传了 `styles = {}`（空对象），导致 `fill` 始终为黄色。SVG 属性优先级高于 CSS class，所以即使添加了自定义 CSS 类名，颜色仍然被硬编码的黄色覆盖。

**渲染链路**：
```
annotations.add(type, cfi, data, cb, className, styles)
  → view.highlight(cfi, data, cb, className, styles)
    → Object.assign({fill:'yellow',...}, styles)  ← styles={} 导致 fill 始终是 yellow
      → Highlight 构造器 → attributes = {fill:"yellow"}
        → element.setAttribute("fill", "yellow")  ← SVG 属性直接设为黄
```

**修复方案**：
在 `useHuaXianChuTi.ts` 的第6个参数（styles）中传入对应颜色的 `{fill}` 值：

| 颜色 | className | styles.fill |
|------|-----------|-------------|
| 黄色 | hl-yellow | #fef08a |
| 绿色 | hl-green | #86efac |
| 蓝色 | hl-blue | #93c5fd |
| 粉色 | hl-pink | #f9a8d4 |

**修改文件**：
- `src/features/books/hooks/useHuaXianChuTi.ts` - 新增 styleMap，两处 add() 调用传入 fill 颜色值

**验证结果**：
- ✅ npm run build 构建成功

### EPUB 滚轮翻页闪烁问题（2026-04-01）

**问题描述**：滚轮翻页时页面出现白屏闪烁，快速滚动时尤为明显。用户反馈为"内容跳动，没有出现平移翻页动画"。

**根本原因：epubjs 底层渲染机制**

epubjs 在翻页时会**先清除旧内容，再渲染新内容**，这是 iframe 渲染的固有行为，无法从应用层改变。白屏发生在旧内容清除后、新内容渲染前的间隙。

**已尝试但失败的方案**：
1. ❌ 添加 500ms 节流 - 节流只能减少触发频率，无法阻止单次翻页时的白屏
2. ❌ 翻页锁 + locationChanged 解锁 - locationChanged 触发时机早于渲染完成
3. ❌ 翻页锁 + rendered 解锁 - rendered 事件触发时白屏已经发生
4. ❌ CSS 淡出/淡入过渡动画 - 淡出后 rendered 事件中的 container 引用是闭包旧值，导致无法淡入，白屏持续
5. ❌ 移除淡出，只保留翻页锁 - 翻页锁无法阻止单次翻页时的白屏

**建议尝试的新方向**：
- **加载遮罩层**：翻页前显示遮罩，渲染完成后隐藏，彻底避免白屏
- **双缓冲预渲染**：预渲染下一页，翻页时直接切换（实现复杂）
- **禁用滚轮翻页**：只保留按钮翻页，避免快速触发

**保留火种（可复用代码）**：
```typescript
// 翻页锁机制（防止连续触发）
const isTurningPageRef = useRef(false);

const handleWheel = (event: WheelEvent) => {
  if (isTurningPageRef.current) return;
  isTurningPageRef.current = true;
```

5. `iframeDoc._wheelHandlerAdded = true` 阻止了事件监听器的更新

### 高亮删除后视觉高亮不消失问题（2026-04-03）

**问题描述**：删除高亮记录后，章节中的视觉高亮没有立刻消失，需要刷新页面才能消失。

**根本原因分析**：

问题出在 `useHuaXianChuTi` Hook 中的高亮应用逻辑。`highlights` 状态使用 `useLocalStorageState` 管理，当删除高亮时，状态确实更新了，但 `useEffect` 中绑定的事件回调使用的是闭包中的旧值。

**已尝试但失败的方案**：

| 尝试次数 | 方案描述 | 失败原因 |
|----------|----------|----------|
| 第1次 | 只在 handleDeleteHighlight 中删除 localStorage 数据 | 只删除了数据，没有同步更新章节中的视觉高亮 |
| 第2次 | useEffect 依赖 [highlights, renditionRef]，highlights 变化时重新应用 | useEffect 可能不会在 highlights 变化时立即触发，或者闭包捕获的是旧值 |
| 第3次 | useEffect 不依赖 highlights，改为直接读取 localStorage | 依赖 rendered 事件触发，但删除操作后事件未触发或时机不对 |
| 第4次 | 使用 setTimeout 延迟调用 rendition.annotations.remove(cfiRange, 'highlight') | 视觉高亮仍未消失，cfiRange 可能已失效或格式不匹配 |
| 第5次 | 参考其他项目使用 annotations.remove(cfi, type) + rendition.display() | rendition.display() 导致页面跳转，无法使用 |
| 第6次 | requestAnimationFrame 延迟重新应用高亮 | 视觉高亮仍未消失 |
| 第7次 | 查找 iframe DOM 操作 - .epubjs-hl、data-cfi 等选择器 | iframe 内找不到任何高亮相关 DOM 元素，epubjs 可能使用虚拟渲染 |
| 第8次 | 通过 rendition.themes.default() 刷新样式 | 视觉高亮仍未消失 |
| 第9次 | 通过 rendition.resize() 强制重绘 | 视觉高亮仍未消失 |

**核心矛盾分析**：

### EPUB 划线高亮颜色全黄问题（2026-04-02）- ✅ 已解决（2026-04-04）

**问题描述**：选中文字设置高亮后，无论选择什么高亮颜色，显示出来的都是黄色。

**根本原因分析**：

epub.js 的 `iframe.js` 第606行 `highlight()` 方法中：
```javascript
const attributes = Object.assign({"fill": "yellow", ...}, styles);
```
我们传了 `styles = {}`（空对象），导致 `fill` 始终为黄色。SVG 属性优先级高于 CSS class。

**修复方案**：

在 `annotations.add()` 第6个参数 `styles` 中传入对应颜色的 `{fill}` 值，覆盖默认黄色。详见上方「高亮颜色多色支持修复」记录。
