## 一、应用启动流程

```
用户打开网页
    ↓
main.tsx（入口文件）加载
    ↓
AppProvider.tsx 初始化全局状态
    ↓
检查用户登录状态（Go后端JWT认证）
    ↓
├── 未登录 → 显示登录页面（AuthPage.tsx）
└── 已登录 → 显示主应用内容
```

**关键文件：**
- `src/core/main.tsx` - 应用入口
- `src/infrastructure/hooks/AppProvider.tsx` - 全局状态管理
- `src/features/user/components/AuthPage.tsx` - 登录页面

## 二、用户登录流程

```
用户输入账号密码
    ↓
点击登录按钮
    ↓
调用 Go 后端认证 API
    ↓

    ↓
├── 成功 → 保存 JWT Token 和用户信息，跳转书架页

```

- `src/shared/services/auth.ts` - 认证服务
- `backend/controllers/auth_login.go` - 登录控制器

### 3.3 AI 学习流程（核心）

本项目聚焦于**主动回忆学习**，而非传统的刷题模式。AI 作为学习伙伴，帮助用户用自己的话表达和理解。

---

### 3.4 名词解释学习

```
用户点击「名词解释学习」（支持章节或段落）
    ↓
检查数据库中是否已保存该来源的概念
    ↓
├── 已保存 → 直接从数据库读取，不调用AI
└── 未保存 → AI从文本中识别重要概念并保存到数据库
    ↓
左侧显示概念列表，右侧显示当前概念详情
    ↓
用户可以自由选择任意概念进行练习：
├── 显示概念名称
├── 显示专业解释
    ↓
用户用自己的话复述概念（口语化表达）
    ↓
AI 评价复述的「表达清晰度」
    ↓
保存练习记录到数据库
```

**AI 角色：**
- 概念提取器：从段落/章节中自动识别重要概念
- 复述评价者：评价用户用自己话复述的质量

**核心特性：**
- ✅ 支持章节和段落两种来源
- ✅ 概念保存到数据库，第二次进入无需重新调用AI
- ✅ 左侧概念列表，右侧详情，支持自由选择
- ✅ 练习记录保存到数据库

**关键文件：**
- `src/features/practice/GaiNianXueXi.tsx` - 名词解释学习组件
- `src/shared/services/aiService.ts` - AI 服务
- `backend/controllers/concepts.go` - 概念提取控制器（带缓存逻辑）
- `backend/controllers/evaluate_concept.go` - 概念点评控制器
- `backend/services/zhipu_concepts.go` - 智谱 AI 概念提取服务
- `backend/services/zhipu_evaluate_concept.go` - 智谱 AI 概念点评服务
- `backend/models/models_content.go` - Concept 和 ConceptPracticeRecord 模型

### 3.5 意图理解学习

```
用户点击「意图理解学习」
    ↓
展示目标段落/章节内容
    ↓
用户用自己的话自由讲述「作者想告诉我们什么」
    ↓
AI 思考特定文本中作者想表达的意思，并以此为标准
    ↓
AI 根据（原文 + 用户回答）从三个维度评价：
├── ✓ 说得对的地方
├── ✗ 说得不对的地方
└── ○ 说得不到位的地方
    ↓
完善用户的口语化回答
```

**AI 角色：**
- 意图分析者：理解作者的核心思想
- 回答完善者：评价并补充用户的自由讲述

**关键文件：**
- `src/features/practice/YiTuLiJie.tsx` - 意图理解学习组件

- `backend/controllers/evaluate_concept.go` - 意图点评控制器

## 六、系统模板内置功能

### 6.1 系统预设模板

系统内置 3 个预设提示词模板，所有用户自动可见：
- **名词解释 - 标准模板**：用于名词解释学习
- **意图理解 - 标准模板**：用于意图理解学习
- **生活应用 - 标准模板**：用于生活应用练习

### 6.2 实现原理

系统模板通过 `user_id = NULL` 标识，查询逻辑为：
```sql
WHERE user_id IS NULL OR user_id = currentUserId
```

这样确保：
- ✅ 新用户注册后自动看到系统模板
- ✅ 所有用户共享同一套系统模板
- ✅ 用户可复制系统模板创建个人自定义版本

### 6.3 初始化方式

通过后端 API 端点初始化系统模板：
```
POST /api/prompts/init-system
```

该接口会：
1. 删除旧的系统模板（`is_system = 1`）
2. 插入 3 个新的系统预设模板

### 6.4 数据模型

```go
type PromptTemplate struct {
    ID           string    `gorm:"type:char(36);primaryKey"`
    UserId       *string   `gorm:"type:char(36);index"`  // NULL = system template
    Name         string    `gorm:"type:varchar(100)"`
    QuestionType string    `gorm:"type:varchar(50)"`
    Content      string    `gorm:"type:text"`
    IsDefault    bool      `gorm:"default:false"`
    IsSystem     bool      `gorm:"default:false"`
}
```

**关键设计**：
- `user_id = NULL`：表示全局共享的系统模板
- `is_system = 1`：标记为系统模板，可被清理重置
- `is_default = true`：标记为默认模板

### 6.5 后端实现

**控制器**：`backend/controllers/prompt_create.go`
```go
// InitSystemPrompts 初始化系统预设模板（临时接口）
func InitSystemPrompts(c *gin.Context) {
    db := config.GetDB()
    
    // 删除旧的系统模板
    db.Where("is_system = ?", 1).Delete(&models.PromptTemplate{})
    
    // 插入 3 个系统预设模板
    // 1. 名词解释 - 标准模板
    // 2. 意图理解 - 标准模板
    // 3. 生活应用 - 标准模板
}
```

**路由**：`backend/routes/routes.go`
```go
prompts := api.Group("/prompts")
prompts.Use(middleware.AuthMiddleware())
{
    prompts.GET("", controllers.GetPromptTemplates)
    prompts.POST("/init-system", controllers.InitSystemPrompts) // 临时接口
}
```

### 6.6 前端使用

前端查询模板时自动包含系统模板：
```typescript
// database.ts - getPromptTemplates
const response = await fetch('/api/prompts', {
  headers: { 'Authorization': `Bearer ${token}` }
})
// 后端返回：系统模板 + 用户个人模板
```

## 八、关键概念解释

### 7.1 什么是 Go 后端？
本项目使用 Go 语言（Gin 框架）构建后端 API：
- **认证**：JWT Token 认证机制
- **数据库**：MySQL 存储用户数据、书籍、章节、段落等
- **AI 服务**：调用智谱 AI 进行概念提取、复述评价、意图分析

### 7.2 什么是智谱AI？
智谱AI（GLM-4）是本项目使用的国产大语言模型，用于：
- 从文本中提取重要概念并给出专业解释
- 评价用户用自己话复述的质量
- 分析作者意图并评价用户的自由讲述
- 完善和补充用户的口语化回答

### 7.3 什么是三层架构？
本项目代码按三层组织：
- **数据逻辑层**：hooks/、services/、api/ - 处理数据和业务逻辑
- **子组件层**：components/business/ - 可复用的UI组件
- **主视图层**：components/*.tsx - 页面主组件

### 8.4 前后端数据格式转换
后端使用 snake_case（如 `book_id`），前端使用 camelCase（如 `bookId`）。
前端服务层负责数据格式转换，确保类型安全。

## 十、常见问题

**Q: 登录失败怎么办？**
A: 检查后端服务是否正常运行，查看 JWT Token 是否有效

**Q: AI不回复怎么办？**
A: 确保后端服务正常运行，内置智谱 AI API Key 已正确配置

**Q: 数据不同步怎么办？**
A: 检查后端 API 是否正常，查看数据库连接状态

**Q: 章节管理功能无法使用？**
A: 确保后端 `/api/chapters` 接口正常工作，检查数据库 chapters 表是否存在
