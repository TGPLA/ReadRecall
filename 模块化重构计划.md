### 实际采用的结构说明

本项目采用简化版模块化结构：

```
src/
├── core/                    # 入口文件
├── infrastructure/          # 类型定义（统一存放）
│   └── types/              # book/, user/, config/ 等
├── features/               # 业务组件（按模块拆分）
│   ├── books/components/   # 书籍相关组件
│   ├── practice/          # 练习组件（无子目录）
│   └── user/components/   # 用户组件
└── shared/                 # 公共服务
    ├── api/               # 外部 API
    ├── services/          # 业务服务
    └── utils/             # 工具函数
```

**设计决策**：
- API 和 Service 统一放在 `shared/`，避免重复
- 类型统一放在 `infrastructure/types/`，便于管理
- 组件按模块拆分，简单模块无需子目录
