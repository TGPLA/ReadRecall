## 十、2026-03-25 数据库外键约束修复

### 问题描述

| 问题 | 错误信息 |
|------|----------|
| 删除用户时报错 | #1451 - Cannot delete or update a parent row: a foreign key constraint fails |
| 约束名称 | `fk_prompt_templates_user` |
| 约束表 | `prompt_templates` |
| 引用表 | `users` |

### 根本原因

| 检查项 | 期望值 | 实际值 | 问题 |
|--------|--------|--------|------|
| 外键 DELETE_RULE | CASCADE | NO ACTION | 删除规则未设置级联 |

虽然数据库迁移脚本中定义了 `ON DELETE CASCADE`，但实际数据库中的外键约束删除规则是 `NO ACTION`，导致删除用户时，如果该用户在 `prompt_templates` 表中有相关记录，数据库会阻止删除操作。

### 修复方案

```sql
-- 1. 删除旧的外键约束
ALTER TABLE prompt_templates DROP FOREIGN KEY fk_prompt_templates_user;

-- 2. 重新添加带 ON DELETE CASCADE 的外键约束
ALTER TABLE prompt_templates
ADD CONSTRAINT fk_prompt_templates_user
FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE;
```

### 验证结果

| 检查项 | 修复前 | 修复后 |
|--------|--------|--------|
| DELETE_RULE | NO ACTION | CASCADE ✅ |
| UPDATE_RULE | NO ACTION | NO ACTION |

### 影响范围

| 表名 | 外键约束 | 删除规则 | 说明 |
|------|----------|----------|------|
| prompt_templates | fk_prompt_templates_user | CASCADE | 删除用户时，自动删除该用户的提示词模板 |

### 修复脚本

已保存修复脚本到：`backend/修复外键约束_prompt_templates.sql`

| 问题 | 原因 |

### 修复方案

#### 1. 健康检查路由修复

在 `backend/routes/routes.go` 添加 `/api/health` 路由：

```go
api.GET("/health", func(c *gin.Context) {
    c.JSON(200, gin.H{
        "status":  "ok",
        "message": "阅读回响后端服务运行正常",
    })
})
```

#### 2. 概念表外键约束修复

在 `backend/models/models_content.go` 移除 Chapter 模型的外键约束：

```go
// 修改前
Concepts []Concept `gorm:"foreignKey:SourceId;constraint:OnDelete:CASCADE" json:"concepts,omitempty"`

// 修改后
Concepts []Concept `gorm:"foreignKey:SourceId" json:"concepts,omitempty"`
```

#### 3. 数据库迁移临时跳过

在 `backend/config/config.go` 暂时跳过自动迁移：

```go

AutoMigrate()

// 修改后
```
