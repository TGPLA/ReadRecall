## 快速部署

### 一键部署脚本

```powershell
# 步骤 1: 构建
npm run build

# 步骤 2: 运行部署脚本
.\deploy.ps1
```

部署脚本执行流程：
1. 检查 dist 目录是否存在
2. 列出待上传文件
3. **清空服务器旧文件**（避免文件堆积）
4. 上传新文件

### 手动部署

#### 1. 构建生产版本

```bash
npm run build
```

构建产物位于 `dist/` 目录。

#### 2. 上传文件到服务器

```powershell
# Windows PowerShell
scp -r dist\* root@114.132.47.245:/opt/1panel/www/sites/readrecall/index/
```

```bash
# Linux/Mac
scp -r dist/* root@114.132.47.245:/opt/1panel/www/sites/readrecall/index/
```

#### 3. 验证部署

访问 https://linyubo.top 确认部署成功。

## Nginx 配置

### 配置文件位置

`/opt/1panel/www/conf.d/readrecall.conf`

### 关键配置说明

```nginx
server {
    listen 80;
    listen 443 ssl;
    server_name linyubo.top;
    
    # SSL 证书配置
    ssl_certificate /www/sites/readrecall/ssl/fullchain.pem;
    ssl_certificate_key /www/sites/readrecall/ssl/privkey.pem;
    
    # 前端静态文件目录
    root /www/sites/readrecall/index;
    
    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
    }
    
    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 配置要点

- **前端目录**：`/www/sites/readrecall/index`
- **API 代理**：`/api/` → `http://127.0.0.1:8080/api/`
- **HTTPS**：已配置 SSL 证书
- **缓存策略**：
  - `index.html` 禁止缓存
  - 带 hash 的静态资源长期缓存（1年）

## 故障排查

### 404 错误

1. 检查文件是否上传到正确目录
2. 检查 Nginx 配置中的 root 路径
3. 检查文件权限

### API 请求失败

1. 检查后端服务是否运行
2. 检查 Nginx 反向代理配置
3. 查看 Nginx 错误日志

### 样式/JS 加载失败

1. 清除浏览器缓存
2. 检查文件是否完整上传
3. 检查 Nginx 缓存配置
