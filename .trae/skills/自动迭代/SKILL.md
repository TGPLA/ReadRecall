# 自动迭代技能 (Auto-Iterate)

## 核心功能

| 功能 | 触发时机 | 自动化程度 |
|------|---------|-----------|
| **任务完成时检查** | 每次 `npm run check:lines` 通过后 | 全自动 |
| **Bug修复后补充** | Bug解决并验证通过后 | 半自动（需确认） |
| **新模块自动扩展** | 创建新组件/Hook/服务文件时 | 全自动 |
| **定期巡检同步** | 每周或每次大版本发布前 | 全自动 |

## 执行流程

```
┌─────────────────────────────────────────────────────────────┐
│                    自动迭代检查触发点                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   触发点A: 任务完成验证                                      │
│   ├── 检查代码变更范围                                       │
│   ├── 识别受影响的技能领域 (L2)                               │
│   └── 更新相关技能的 L3/L4 内容                              │
│                                                             │
│   触发点B: Bug修复归档                                       │
│   ├── 读取复盘存档                                           │
│   ├── 提取解决方案关键词                                      │
│   └── 补充到对应技能的速查表                                  │
│                                                             │
│   触发点C: 新文件创建检测                                     │
│   ├── 扫描 src/ 目录增量                                     │
│   ├── 识别文件类型 (组件/Hook/服务/类型)                       │
│   └── 更新模块导航表                                         │
│                                                             │
│   触发点D: 定期巡检                                          │
│   ├── 全量扫描代码库                                         │
│   ├── 校验技能文档与实际代码的一致性                           │
│   └── 生成差异报告                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 核心脚本 (迭代引擎)

### 1. 增量扫描脚本 (`迭代引擎.js`)

**位置**: `.trae/skills/自动迭代/迭代引擎.js`

```javascript
/**
 * 自动迭代引擎 - 阅读回响技能体系
 *
 * 功能：
 * 1. 扫描代码增量，检测新文件/修改文件
 * 2. 识别文件类型，更新模块导航表
 * 3. 提取Bug解决方案，补充速查表
 * 4. 校验技能文档一致性
 */

const fs = require('fs')
const path = require('path')

// ==================== 配置区 ====================

const 项目根目录 = path.join(__dirname, '../../../')
const 技能目录 = path.join(项目根目录, '.trae/skills')
const 复盘存档目录 = path.join(项目根目录, 'archived_复盘存档')

// 扫描范围配置
const 扫描配置 = {
  前端: {
    根目录: path.join(项目根目录, 'src'),
    文件模式: [
      { 目录: 'features/books/components', 类型: '组件', 后缀: '.tsx' },
      { 目录: 'features/books/hooks', 类型: 'Hook', 后缀: '.ts' },
      { 目录: 'features/practice', 类型: '组件', 后缀: '.tsx' },
      { 目录: 'features/user/components', 类型: '组件', 后缀: '.tsx' },
      { 目录: 'shared/services', 类型: '服务', 后缀: '.ts' },
      { 目录: 'shared/utils', 类型: '工具', 后缀: '.ts' },
      { 目录: 'infrastructure', 类型: '基础设施', 后缀: '.ts' },
    ]
  },
  后端: {
    根目录: path.join(项目根目录, 'backend'),
    文件模式: [
      { 目录: 'controllers', 类型: '控制器', 后缀: '.go' },
      { 目录: 'services', 类型: '服务', 后缀: '.go' },
      { 目录: 'models', 类型: '模型', 后缀: '.go' },
      { 目录: 'middleware', 类型: '中间件', 后缀: '.go' },
    ]
  }
}

// ==================== 核心功能 ====================

/**
 * 扫描目录获取所有匹配文件
 */
function 扫描目录(根目录, 文件模式) {
  const 结果 = []

  for (const 模式 of 文件模式) {
    const 完整路径 = path.join(根目录, 模式.目录)
    if (!fs.existsSync(完整路径)) continue

    const 文件列表 = fs.readdirSync(完整路径, { withFileTypes: true })
    for (const 文件 of 文件列表) {
      if (文件.isFile() && 文件.name.endsWith(模式.后缀)) {
        结果.push({
          名称: path.basename(文件.name, 模式.后缀),
          完整路径: path.join(完整路径, 文件.name),
          类型: 模式.类型,
          修改时间: fs.statSync(path.join(完整路径, 文件.name)).mtime
        })
      }
    }
  }

  return 结果.sort((a, b) => b.修改时间 - a.修改时间)
}

/**
 * 生成模块导航表Markdown
 */
function 生成导航表(标题, 文件列表, 技能类型) {
  let md = `### ${标题}\n\n`
  md += `**路径**: \`${文件列表[0]?.完整路径?.replace(/\\/[^/]+\/[^/]+$/, '') || 'N/A'}\`\n\n`
  md += `| 名称 | 类型 | 说明 |\n`
  md += `|------|------|------|\n`

  for (const 文件 of 文件列表) {
    const 中文类型 = {
      '组件': 'Component',
      'Hook': 'Hook',
      '服务': 'Service',
      '工具': 'Utils',
      '控制器': 'Controller',
      '模型': 'Model',
      '中间件': 'Middleware',
      '基础设施': 'Infra'
    }[文件.类型] || 文件.类型

    const 说明 = 文件.名称
      .replace(/([A-Z])/g, ' $1')
      .replace(/^./, str => str.toUpperCase())
      .replace(/[_-]/g, ' ')
      .trim()

    md += `| \`${文件.名称}\` | ${中文类型} | ${说明} |\n`
  }

  return md
}

/**
 * 检测代码增量
 */
function 检测增量(上次扫描时间戳) {
  const 增量 = {
    新文件: [],
    修改文件: [],
    删除文件: []
  }

  // 扫描前端
  for (const [领域, 配置] of Object.entries(扫描配置)) {
    const 文件列表 = 扫描目录(配置.根目录, 配置.文件模式)
    for (const 文件 of 文件列表) {
      if (文件.修改时间 > 上次扫描时间戳) {
        增量.修改文件.push(文件)
      }
    }
  }

  return 增量
}

/**
 * 从复盘存档提取解决方案
 */
function 提取解决方案() {
  if (!fs.existsSync(复盘存档目录)) return []

  const 存档文件 = fs.readdirSync(复盘存档目录)
    .filter(f => f.endsWith('.md'))
    .map(f => ({
      名称: f,
      路径: path.join(复盘存档目录, f),
      时间: fs.statSync(path.join(复盘存档目录, f)).mtime
    }))
    .sort((a, b) => b.时间 - a.时间)

  return 存档文件.slice(0, 5).map(存档 => {
    const 内容 = fs.readFileSync(存档.路径, 'utf-8')
    const 问题匹配 = 内容.match(/## (问题|症状|Bug)[^\n]*\n([^\n]+)/)
    const 方案匹配 = 内容.match(/## (解决方案|修复)[^\n]*\n([\s\S]+?)(?=##|$)/)

    return {
      来源: 存档.名称.replace('.md', ''),
      问题: 问题匹配 ? 问题匹配[2].trim() : '未知',
      解决方案: 方案匹配 ? 方案匹配[2].trim().slice(0, 200) : '待提取',
      关键词: 提取关键词(内容)
    }
  })
}

/**
 * 提取内容关键词
 */
function 提取关键词(内容) {
  const 关键模式 = [
    /划线|标注|highlight/i,
    /翻页|page|epub/i,
    /上传|upload|导入/i,
    /登录|认证|auth/i,
    /数据库|sql|mysql/i,
    /部署|docker|nginx/i,
    /测试|test|vitest|playwright/i
  ]

  return 关键模式
    .filter(模式 => 模式.test(内容))
    .map(模式 => 模式.toString().replace(/[\/\(\)\|]/g, '').replace('i', ''))
}

/**
 * 主迭代函数
 */
async function 执行迭代(选项 = {}) {
  console.log('🔄 开始技能迭代...')

  const 结果 = {
    时间: new Date().toISOString(),
    增量: { 新文件: [], 修改文件: [] },
    解决方案: [],
    同步状态: 'unknown'
  }

  // 1. 扫描增量
  const 上次时间 = 选项.上次扫描时间 || new Date(0)
  const 增量 = 检测增量(上次时间)
  结果.增量 = 增量

  console.log(`📊 发现 ${增量.修改文件.length} 个修改文件`)

  // 2. 提取解决方案
  if (选项.包含复盘) {
    结果.解决方案 = 提取解决方案()
    console.log(`📝 提取 ${结果.解决方案.length} 个复盘解决方案`)
  }

  // 3. 生成报告
  const 报告 = 生成迭代报告(结果)
  console.log('\n📋 迭代报告:\n', 报告)

  return 结果
}

/**
 * 生成迭代报告
 */
function 生成迭代报告(结果) {
  return {
    修改文件数: 结果.增量.修改文件.length,
    解决方案数: 结果.解决方案.length,
    建议操作: 结果.增量.修改文件.length > 0 ? '更新模块导航表' : '无需操作',
    详细: 结果
  }
}

// ==================== 导出 ====================

module.exports = {
  扫描目录,
  检测增量,
  提取解决方案,
  执行迭代,
  生成导航表
}

// CLI 入口
if (require.main === module) {
  const 选项 = {
    上次扫描时间: process.argv[2] ? new Date(process.argv[2]) : new Date(Date.now() - 7 * 24 * 60 * 60 * 1000),
    包含复盘: true
  }
  执行迭代(选项).then(结果 => {
    console.log('\n✅ 迭代完成')
    process.exit(0)
  }).catch(错误 => {
    console.error('❌ 迭代失败:', 错误)
    process.exit(1)
  })
}
```

## 触发规则

### 规则A: 任务完成自动触发

**触发条件**: 满足以下任一条件时
- `npm run build` 构建成功
- `npm run check:lines` 检查通过
- 用户明确说"任务完成"或"可以提交了"

**执行动作**:
```
1. 询问: "是否运行自动迭代检查？"
2. 如果用户同意，运行迭代引擎
3. 根据检测结果更新相关技能文件
4. 生成差异报告
```

### 规则B: Bug修复自动触发

**触发条件**: 满足以下任一条件时
- 用户提到"修复了Bug"或"解决了问题"
- 复盘存档目录出现新文件
- 用户要求"更新技能"

**执行动作**:
```
1. 扫描 archived_复盘存档/ 目录
2. 读取最新的复盘文件
3. 识别问题类型和解决方案
4. 更新对应技能的速查表
5. 标记已同步的复盘文件
```

### 规则C: 新文件创建自动触发

**触发条件**: 满足以下任一条件时
- 创建了新的 `.tsx` 或 `.ts` 文件
- 创建了新的 `.go` 文件
- 用户提到"新建了组件"或"添加了新模块"

**执行动作**:
```
1. 识别新文件的类型和位置
2. 确定归属的 L2 技能领域
3. 更新该技能的模块导航表
4. 在对应 L3 下添加新条目
```

### 规则D: 定期巡检

**触发条件**: 满足以下任一条件时
- 每周一早上（自动提醒）
- 大版本发布前
- 用户说"巡检"或"同步检查"

**执行动作**:
```
1. 全量扫描代码库
2. 对比技能文档中的模块列表
3. 检测以下不一致:
   - 技能中记录的文件不存在
   - 存在但技能中未记录的文件
   - 文件名/路径变更
4. 生成同步差异报告
5. 自动修复或列出待修复项
```

## 同步策略

### 优先级规则

| 优先级 | 内容类型 | 同步策略 |
|-------|---------|---------|
| P0 | Bug解决方案 | 立即同步，24小时内 |
| P1 | 新组件/新模块 | 任务完成后同步 |
| P2 | 代码重构（改名/移动） | 同步时附带旧路径备注 |
| P3 | 文档修正 | 每周汇总同步 |

### 冲突处理

当技能文档与实际代码不一致时：

```
1. 提示用户选择保留策略:
   - [A] 以代码为准 (更新技能)
   - [B] 以技能为准 (回退代码)
   - [C] 人工审核 (列出差异让用户决定)

2. 默认策略: 以代码为准 (选项A)
   - 代码是最终事实，技能文档应跟随代码

3. 记录变更历史
   - 在技能文件头部添加变更日志
   - 格式: ## YYYY-MM-DD 更新 - [变更内容]
```

## 注意事项

⚠️ **自动迭代的限制**:
1. **不自动删除**: 只添加和修改，不自动删除技能内容
2. **不强制更新**: 所有变更需用户确认
3. **保留变更历史**: 每次更新都记录时间戳和内容

⚠️ **触发频率限制**:
- 单次对话中最多触发 2 次自动迭代
- 防止频繁打断用户工作流

⚠️ **安全边界**:
- 只修改 `.trae/skills/` 目录下的文件
- 不修改 `src/` 或 `backend/` 的源代码
- 不提交未经用户确认的变更
