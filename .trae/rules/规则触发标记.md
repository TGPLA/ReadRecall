# 规则触发标记使用说明

## 方案概述

通过在代码中添加特定注释标记，自动统计各规则的触发次数。

## 标记格式

```typescript
// @规则触发:规则名称[可选描述]
```

## 示例

```typescript
// 创建文件时
// @规则触发:文件操作规范[创建组件时检查行数限制]
const XinZuJian = () => { ... };

// 验证功能时
// @规则触发:任务与功能验证规范[完成登录功能后验证UI入口]
if (!uiRuKou) {
  throw new Error('UI 控制入口缺失');
}

// 调试时
// @规则触发:前端调试规范[检查控制台错误]
console.error('运行时错误:', error);
```

## 统计命令

```bash
# 查看统计报告
cd .trae/rules && node 规则触发统计.cjs report

# 手动记录一次触发
cd .trae/rules && node 规则触发统计.cjs record "规则名称" "描述"

# 导出 CSV 报告
cd .trae/rules && node 规则触发统计.cjs export

# 扫描代码中的标记
cd .trae/rules && node 扫描规则标记.cjs [文件或目录路径]
```

## 支持的规则名称

- 任务与功能验证规范
- 文件操作规范
- 全局语言规范
- 模式优化护栏
- 前端调试规范
- 后端API集成测试规范
- 数据库服务器自动连接规范
- 目录使用规范
- 架构变更规范
- 版本号管理规范

## 数据存储

统计数据保存在 `规则触发统计.json` 文件中，格式如下：

```json
{
  "version": "1.0",
  "lastUpdated": "2026-04-14T...",
  "rules": {
    "任务与功能验证规范": {
      "count": 5,
      "triggers": [
        { "timestamp": "...", "context": "..." }
      ]
    }
  }
}
```

## 自动化建议

可以在 `.git/hooks/pre-commit` 中添加：

```bash
#!/bin/bash
echo "扫描规则触发标记..."
node .trae/rules/扫描规则标记.cjs
```

这样每次提交前自动扫描代码中的标记并更新统计。
