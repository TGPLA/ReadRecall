# 文件操作规范

## 修改文件前

1. **关键代码检查**（修改核心逻辑时）
   - 搜索 `@关键代码` 标记，查看 `.trae/关键代码清单.md`
   - 理解原因后再修改，修改后验证功能
2. 查阅 `02_项目地图.md` 确认文件定位
3. 检查是否有 `// @审计已完成` 标记
4. 确认修改后行数 ≤ 100 行

## 删除文件前

1. **关键代码检查**（必须）
   - 搜索 `@关键代码` 标记，查看清单，理解原因
2. 全局搜索确认无引用
3. 物理删除，同步更新 `02_项目地图.md`【归档区】

## 创建文件时

1. 中文头部注释 + `// @审计已完成` 标记
2. 行数 ≤ 100 行，超则按三层架构拆分
3. **无论来源**（复制/参考/AI生成）都必须遵守规则
4. **新旧代码**：新代码产生时询问是否删除旧代码

## 命名规范

参见 [全局语言规范.md](./全局语言规范.md)：
- 文件名：中文拼音（如 `ShuJiTiShiCiPeiZhi.ts`）
- 函数/变量名：中文（如 `huoQuPeiZhi`）
- 注释：中文

## 检查命令

```bash
npm run check:lines  # 检查行数
npm run build        # 构建验证
```
